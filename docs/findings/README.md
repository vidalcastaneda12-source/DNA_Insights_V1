# Findings

Short documents capturing real-data observations and architectural decisions
that emerged during development. Each finding answers: what did we observe,
why does it matter, and what (if anything) follows from it.

These are durable records of discoveries that might otherwise live only in
chat history or git commit messages. They are not designed documents (those
live in `docs/schemas/`) or runbooks (those live in `docs/runbooks/`) — they
capture the *why* behind specific choices, not the *what* or the *how*.

Findings are numbered for stable reference (`finding-001-...`). New findings
should follow the established format: a single H1 title, a short context
section, the observation, the implication, and any follow-up tracking.
