# Findings

Short documents capturing real-data observations and architectural decisions
that emerged during development. Each finding answers: what did we observe,
why does it matter, and what (if anything) follows from it.

These are durable records of discoveries that might otherwise live only in
chat history or git commit messages. They are not designed documents (those
live in `docs/schemas/`) or runbooks (those live in `docs/runbooks/`) — they
capture the *why* behind specific choices, not the *what* or the *how*.

Findings are numbered for stable reference (`finding-001-...`). New findings
should follow the established format: a single H1 title, a short context
section, the observation, the implication, and any follow-up tracking.

## Frontmatter convention

Every finding begins with a `---`-fenced frontmatter block **above** its `# Finding NNN`
H1, parsed and enforced by `genome docs check` (the decision-tracking gate, finding-036).
The block is a flat key set — a minimal stdlib parser, not PyYAML. Keys:

- `type` — `observation` | `decision` | `both`. A `decision`/`both` finding **must** have a
  corresponding `DEC-NNNN` row in the root `MEMORY.md` ledger (the gate fails otherwise).
- `status` — lifecycle: `active` | `superseded` | `reversed` | `deferred`. (Distinct from a
  decision's *kind* `architectural` | `tactical`, which lives on the ledger row.)
- `actors` — a list of the five canonical names (`ClaudeCodeVerification`,
  `ClaudeCodeTestingBugs`, `ClaudeCodePlanning`, `ClaudeCodeDevelopment`, `VSC-User`); legacy
  spellings (`VSC-Claude`, …) are mapped on read.
- `date` — ISO-8601 `YYYY-MM-DD`.
- `supersedes` / `superseded_by` — lists of finding-ids or DEC-ids. **Frontmatter is the
  authoritative source of the supersession edge**; the index cross-links below are *derived*.

Example block (above the H1):

```text
---
type: decision
status: superseded
actors: [ClaudeCodeDevelopment]
date: 2026-05-22
supersedes: []
superseded_by: [finding-035]
---
# Finding 011 — ...
```

Regenerate the index below with `genome docs build-index`; validate with `genome docs check`.
